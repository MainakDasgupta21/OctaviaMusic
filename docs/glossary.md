# Glossary

> **What you'll learn here:** plain-language definitions of the jargon, abbreviations, and Octavia-specific terms used across the codebase and these docs. When a word trips you up, look here.

---

## Project & product terms

**Octavia** — The product name of this music streaming app (regardless of the repo folder `harmony-hub` or the frontend package name `vite_react_shadcn_ts`).

**Catalog** — The pool of music metadata (songs, albums, artists). In Octavia it's fetched live from external providers and cached — never stored in the database.

**Fallback catalog** — A small, built-in static dataset (`server/data/catalog.js`) used to populate search/detail/charts when a live source fails, so the UI never goes blank.

**Smart queue** — A play queue that auto-fills with similar/radio tracks after you start a song, so playback continues without manual queueing. Built from `getExploreSimilar` + `getExploreRadio`.

**Now Playing** — The full-screen `/player` view (vinyl artwork, transport, lyrics, queue), as opposed to the small footer player.

**Footer player** — The persistent mini-player bar at the bottom of the screen that holds the actual (hidden) YouTube player and keeps audio going across navigation.

**Rail** — A horizontally-scrolling row of cards (e.g. trending, fresh finds) on the Home/Explore pages.

**Explore / Discovery** — The recommendation experience: mood boards, swipe decks, journeys, daily mixes, infinite "flow". Gated by `VITE_EXPLORE_*` feature flags.

**Journey** — A curated, themed listening sequence in Explore (e.g. "night drive", "soft focus").

**Loop / Progression** — The gamification layer (XP, levels, streaks, daily challenges, badges) in Explore. Persisted in `localStorage` via `explore-progression.js`.

**Pulse** — Live/"what's happening" community discovery data (`/api/explore/pulse`).

**Editorial / masthead** — The magazine-style design language: an "issue number", date masthead, eyebrows, hairline rules. Generated by `editorial-meta.js`.

**Spotlight artist** — The weekly featured "artist of the week" on Home, chosen by `spotlight-pick.js`.

**This Day in Music** — An editorial fact card on the charts page (`thisDayInMusic.js`).

## Frontend / React terms

**SPA (Single-Page Application)** — A web app that loads once and updates the page in place via JavaScript instead of full page reloads.

**Component** — A reusable, self-contained piece of UI in React (e.g. `TileCard`, `Sidebar`).

**Props** — Inputs passed from a parent component to a child.

**Hook** — A reusable function (named `use*`) that adds state or behavior to a component (e.g. `usePlayer`, `useChartData`).

**Context / Provider** — React's mechanism for sharing state across many components without passing props down every level. A "provider" wraps the tree and supplies the value; components read it via a hook.

**React Query (TanStack Query)** — The library managing **server state** — fetching, caching, deduping, and revalidating data from the backend.

**Query key** — The array that uniquely identifies a cached query (e.g. `['album', id]`). Same key = shared cache.

**Optimistic update** — Updating the UI/cache immediately as if a mutation succeeded, then reconciling with the server (rolling back on error). Used for favorites, playlists, settings.

**Stale time** — How long React Query considers cached data "fresh" before it's eligible for a background refetch.

**Lazy loading / code splitting** — Loading a page's JavaScript only when you navigate to it (`React.lazy`), keeping the initial bundle small.

**Prefetch** — Loading a route's code/data *before* navigation (e.g. on hover) so the page appears instantly.

**Suspense** — A React feature for showing a fallback (spinner) while a lazy component loads.

**Error boundary** — A component that catches render errors in its subtree and shows a fallback instead of a white screen.

**Route guard** — A wrapper component that redirects based on auth/role (`ProtectedRoute`, `RoleRoute`).

**DTO (Data Transfer Object)** — A clean, frontend-ready data shape. The backend maps messy upstream responses into DTOs so the frontend renders them directly (`server/lib/mappers.js`).

**Design token** — A named value (color, spacing, radius) stored as a CSS variable, so the whole app can be re-themed by changing the token.

**Accent / dynamic accent** — The `--track-accent` color that follows the album art of the current song (or rotates when idle), tinting rings, gradients, etc.

**`cn()`** — Octavia's class-name helper (`src/lib/utils.js`) combining `clsx` + `tailwind-merge` to compose and de-conflict Tailwind classes.

**shadcn/ui** — A pattern where accessible component source (built on Radix) is copied into the repo (`src/components/ui/`) and styled in-house.

**Radix** — The unstyled, accessible UI primitive library underneath shadcn/ui.

**ui-v2** — Octavia's branded design-system components (`src/components/ui-v2/`) built on top of the tokens; preferred over raw `ui/` primitives for product UI.

## Backend / server terms

**Express** — The Node.js web framework serving Octavia's REST API.

**Middleware** — A function that runs on a request before the handler (e.g. auth check, rate limit, validation, CORS).

**Route → Controller → Service → Client/Lib** — The backend's layered flow: routes declare URLs, controllers handle HTTP, services hold logic, clients/libs reach external APIs/DB.

**REST API** — An HTTP API where URLs represent resources and HTTP verbs (GET/POST/PATCH/DELETE) represent actions.

**Mongoose** — The library mapping JavaScript objects to MongoDB documents with schemas/validation.

**Schema / Model** — A Mongoose definition of a collection's shape (`User`, `Playlist`); a Model is the queryable object built from a schema.

**Collection** — MongoDB's equivalent of a table (e.g. `favorites`, `playlists`).

**Index** — A database structure that speeds up lookups; a **unique compound index** also prevents duplicate rows (e.g. one favorite per user+track).

**Snapshot (track snapshot)** — A copy of a track's metadata (title, artist, thumbnail) stored alongside a favorite/playlist so it renders without a live lookup.

**Request coalescing** — Collapsing multiple concurrent identical requests into a single upstream call, sharing the result. Used in the YTM/Last.fm/lyrics clients.

**LRU cache** — "Least Recently Used" — an in-memory cache that evicts the oldest entries when full. Used to cache catalog responses.

**TTL (Time To Live)** — How long a cached value stays valid before it's considered stale/expired.

**Stale-while-revalidate** — Serving slightly-stale cached data immediately while refreshing it in the background.

**Rate limiting** — Capping how many requests a client can make in a time window (returns `429` when exceeded).

**Zod** — The schema-validation library used to validate request bodies/params/queries (and frontend forms).

**Warmup** — The backend pre-fetching charts + trending on startup so the first user request is fast.

## Auth / security terms

**JWT (JSON Web Token)** — A signed token carrying claims (like user id + role) that the server can verify without a database lookup.

**Access token** — A short-lived JWT (≈15m) proving identity on each request.

**Refresh token** — A long-lived JWT (≈30d) used to silently obtain new access tokens.

**Token rotation** — Issuing a new refresh token (and invalidating the old one) every time a refresh happens.

**Reuse detection** — Detecting that an already-rotated refresh token was presented again (a sign of theft) and revoking all sessions in response.

**HttpOnly cookie** — A cookie JavaScript can't read, protecting tokens from XSS.

**CSRF (Cross-Site Request Forgery)** — An attack where another site triggers an authenticated request on your behalf. Octavia defends with a CSRF token echoed in the `x-csrf-token` header.

**Double-submit token** — A CSRF defense where the same token is sent both as a cookie and a header, and the server checks they match.

**bcrypt** — A deliberately slow password-hashing algorithm; "rounds" controls the cost.

**`jti`** — "JWT ID" — a unique identifier per token, used to track/revoke individual refresh sessions.

**Bearer token** — A token sent in the `Authorization: Bearer <token>` header (an alternative to cookies; CSRF checks are skipped for these).

## Music / data terms

**`videoId`** — The YouTube video identifier the browser uses to stream a track's audio. The single most important field the backend returns.

**YouTube Music (`ytmusic-api`)** — The unofficial client providing Octavia's catalog (search, albums, artists, thumbnails, `videoId`s).

**Last.fm** — The API providing chart rankings, play counts, and similar-track data (needs `LASTFM_API_KEY`).

**MusicBrainz** — The open music encyclopedia providing artist country and release dates.

**LRCLib** — The free database providing (time-synced) lyrics.

**LRC** — A lyrics file format with timestamps so each line can highlight in sync with playback (`src/lib/lrc.js`).

**Slug** — A URL-safe identifier derived from a name (e.g. an artist name → `daft-punk`) used in routes like `/artist/:slug` (`src/lib/slug.js`).

**Window (chart window)** — The chart time range: `today`, `this_week`, `this_month`, `all_time`.

**Region** — The chart geography: `global`, `india`, `us`, `uk`, `japan`.

## Tooling terms

**Vite** — The build tool + dev server for the frontend.

**HMR (Hot Module Replacement)** — Vite updating changed modules in the browser without a full reload.

**Vitest** — The test runner (Vite-native, Jest-like).

**Testing Library** — A testing approach that interacts with components the way a user would.

**ESLint** — The linter enforcing code quality/style.

**Monorepo** — One Git repository holding multiple apps (here: frontend at root + backend in `server/`).

**Feature flag** — An env-driven toggle (`VITE_EXPLORE_*`) that turns parts of the UI on/off without code changes.
